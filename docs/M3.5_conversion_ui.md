# M3.5 — Conversion UI (Bridging Work)

## Context

This work was implemented between M3 (Structured Capture) and the official M4
(AI Assistant). It adds the frontend flows that let visitors submit orders,
donations, and pledges through the storefront, plus admin pages for managing
them. Git commits are tagged `feat(m4)` but the official M4 milestone is the
AI Assistant per `backlog-v1.md`.

## What Was Built

### Frontend — Public Storefront
- **Visit hook** (`hooks/use-visit.ts`): creates/reuses a visit per slug via
  `POST /storefront/{slug}/visit`, captures UTM params from URL, persists
  `visit_id` in localStorage.
- **Cart hook** (`hooks/use-cart.ts`): in-memory cart with sessionStorage
  persistence, keyed by slug. Upserts by `catalogItemId`.
- **Storefront page** (`storefront/[slug]/page.tsx`): "Add to Cart" buttons on
  product cards, cart badge in header, Donate/Pledge nav links.
- **Checkout page** (`storefront/[slug]/checkout/page.tsx`): cart summary with
  qty controls, customer form, calls `POST /storefront/{slug}/orders` with
  `visit_id`.
- **Donate page** (`storefront/[slug]/donate/page.tsx`): donation form, calls
  `POST /storefront/{slug}/donations` with `visit_id`.
- **Pledge page** (`storefront/[slug]/pledge/page.tsx`): pledge form with
  future-date picker, calls `POST /storefront/{slug}/pledges` with `visit_id`.

### Frontend — Admin Dashboard
- **Orders list** (`dashboard/orders/page.tsx`): table with status badges and
  transition buttons (confirmed/cancelled/fulfilled).
- **Donations list** (`dashboard/donations/page.tsx`): table with transition
  buttons (received/receipted/cancelled).
- **Pledges list** (`dashboard/pledges/page.tsx`): table with transition buttons
  (partially_fulfilled/fulfilled/lapsed).
- **Dashboard home** (`dashboard/page.tsx`): added "Transactions" section with
  links to orders/donations/pledges.

### Backend — Admin List Endpoints
- `GET /api/v1/tenants/me/orders` — limit/offset, optional `?status=` filter
- `GET /api/v1/tenants/me/donations` — same pattern
- `GET /api/v1/tenants/me/pledges` — same pattern
- Role: member+. Tenant-scoped via `get_db_with_tenant` + RLS.
